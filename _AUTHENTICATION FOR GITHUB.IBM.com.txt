STEPS TO SET UP AUTHENTICATION FOR GITHUB.IBM.COM


Servers
Where the API lives
(e.g., https://github.ibm.com/api/v3)
Paths
Every REST endpoint defined formally
(e.g., /repos/{owner}/{repo}/readme)
Parameters
What each endpoint accepts
(e.g., path params, query params)
Responses
Machine-readable response schemas
(e.g., JSON fields, types)
Security schemes
Token requirements (bearerAuth)
Tags
Groups endpoints in UI (Docs, Search, Repository Info)
Reusable schemas
Under components

For bearerAuth, go to your profile on  GITHUB.IBM.COM -> settings -> developer settings -> personal access token -> tokens(classic)/(fine grained)-> select scopes-> generate token


 
  
  



Go to watson-orchestrate.cloud.ibm.com-> from left side bar-> build-> create agent ->Add tool-> upload OpenAPI file -> Upload files-> continue -> select operations -> connect -> Add new item -> provide “connection Id” & “Display name” -> save and continue -> configure draft connection -> choose an authentication type (bearer token) (ADD THE TOKEN 
(generated from github.ibm.com) -> select member credential/ team credential

If Member credential: Agent will ask every user to input their personal tokens
If Team credential: One credential you can provide after clicking on Team credential-> select Connect.
